## 📋 1. การจัดการพลังงานและ WiFi บนเฟิร์มแวร์ (Firmware & Power)
> [!NOTE]  
> วัตถุประสงค์เพื่อทดสอบความประหยัดพลังงานและการเชื่อมต่อที่แม่นยำของบอร์ด ESP32-C3

### 🟩 [ ] 1.1 WiFi Off on Startup (ปิดไวไฟตอนบูต)
* **วิธีการทดสอบ:** เปิดสวิตช์จ่ายไฟให้บอร์ดตามปกติในสภาวะเริ่มต้น
* **ผลที่คาดหวัง:** บอร์ดต้องบูตเข้าลูปทำงานโดย **ไม่มี** การเชื่อมต่อ WiFi หรือพยายามสแกนหาเครือข่ายใดๆ ทันที
* **สถานะการทดสอบ:** 
  - [ ] ผ่าน (WiFi ปิดสนิทตอนเริ่มต้น)
  - [ ] ไม่ผ่าน
* **บันทึก:** __________________________________________________________________

### 🟩 [ ] 1.2 Test Mode Auto-Disconnect (ตัดไวไฟอัตโนมัติ)
* **วิธีการทดสอบ:** กดสวิตช์เข้าสู่โหมดทดสอบ (Test Mode) บอร์ดต่อ WiFi และส่งไฟล์เสียงขึ้นเซิร์ฟเวอร์
* **ผลที่คาดหวัง:** หลังจากอัปโหลดเสร็จสิ้น บอร์ดต้องเรียกคำสั่ง `wifi_disconnect()` และปิดโมดูล WiFi ลงทันที
* **สถานะการทดสอบ:** 
  - [x] ผ่าน (ตัด WiFi และปิดโหมด STA ทันทีหลังอัปโหลดสำเร็จ)
  - [ ] ไม่ผ่าน
* **บันทึก:** __________________________________________________________________

### 🟩 [ ] 1.3 Normal Mode Batch Upload (ส่งไฟล์แบบกลุ่มรอบ 10 นาที)
* **วิธีการทดสอบ:** ไอใส่เครื่อง 2-3 ครั้งใน Normal Mode (บอร์ดบันทึกลง Flash) แล้วปล่อยเครื่องทิ้งไว้จนครบรอบ 10 นาที
* **ผลที่คาดหวัง:** เมื่อครบรอบบอร์ดต้องเชื่อมต่อ WiFi -> ส่งไฟล์ที่ค้างทั้งหมดในแฟลช -> เมื่อส่งครบหมดคิวแล้ว ต้องตัดการเชื่อมต่อและปิด WiFi ทันที
* **สถานะการทดสอบ:** 
  - [ ] ผ่าน (ส่งไฟล์ครบชุดและปิด WiFi ทันทีหลังจบกระบวนการ)
  - [ ] ไม่ผ่าน
* **บันทึก:** __________________________________________________________________

### 🟩 [ ] 1.4 FIFO Flash Management (ระบบคิวจัดการแฟลชเต็ม)
* **วิธีการทดสอบ:** อัดเสียงต่อเนื่องหลายๆ ครั้งจนแฟลชเต็ม (หรือตั้งค่า Limit ชั่วคราวให้ต่ำลง)

* **สถานะการทดสอบ:** 
  - [ ] ผ่าน (ระบบ FIFO ทำงานปกติ บอร์ดบันทึกต่อได้ไม่ค้าง)

## 🔋 2. การทดสอบระยะเวลาการใช้งานแบตเตอรี่ (Battery Run-time)
> [!TIP]  
> แบตเตอรี่ทดสอบขนาด **520 mAh** การใช้พลังงานที่ต่ำจะช่วยยืดอายุการทำงานของอุปกรณ์

### 🟩 [ ] 2.1 Battery Standby Duration (ทดสอบความอึดแบตเตอรี่)
* **วิธีการทดสอบ:** ชาร์จแบตเตอรี่ 520 mAh จนเต็ม 100% เปิดสวิตช์เริ่มใช้งานใน Normal Mode ทิ้งไว้แบบธรรมชาติ
* **ผลที่คาดหวัง:** บอร์ดสามารถอยู่รอดและเก็บข้อมูลประวัติการไอได้นานขึ้นอย่างเห็นได้ชัดเมื่อเทียบกับเฟิร์มแวร์เวอร์ชันเก่า
* **สถานะการทดสอบ:**
